JSONPath

JSONPath is referred to commonly throughout this documentation. It is a string that is used to query objects at a number of points within the flow. A query is run on a JSON object and will return an array of matches.

More details can be found here: https://www.npmjs.com/package/jsonpath

Query examples

This example shows a potential object which could be queried by JSONPath. The following examples show a selector, and the object which would be resolved by the Flow when performing the query.

// object being queried
var object = {
foo: {
bar: "baz"
}
};

Root object

Returns all matches for the value of the whole object.

"$" // [{foo: { bar: "baz"} }]

Property

Returns all matches for the value of the property “foo” on the root object.

"$.foo" // [{ bar: "baz" }]

Nested property

Returns all matches for the value of bar on the property foo of the root object.

"$.foo.bar" // ["baz"]

Non-existent property

Returns all matches for the value of property bar on the root object. The root object does not contain that property so nothing is matched.

"$.bar" // []

In these examples only up to a single result is returned, however there are times when more complex selectors will be used, and multiple results are expected.

In the majority of cases only a single value will be required. JSONPath in API Builder Flows supports an extension to the suffix of a query, allowing a value at a certain index to be selected from the array of matches. Almost all the time 0 will be the index, selecting the first value matched since only one was expected.

JSONPath Selector

Description

|

Index operator

Example

This shows a potential object which could be queried by JSONPath. The following examples show a selector, and the object which would be resolved by the Flow when performing the query.

// object being queried
var object = {
foo: {
bar: "baz"
},
foo2: {
qux: "quux"
}
};

This example shows the first result of the query for the value of foo.bar.

"$.foo.bar|0" // "baz"

This example uses * which selects all properties. This is being performed on the root object, so both the values of foo and foo2 will be matched. The index operator is used to select the first match.

"$.*|0" // {bar: "baz"}

Similarly in this case, the index operator is used to select the second match.

"$.*|1" // {qux: "quux"}

In this case, baz does not exist, so there was no match. Selecting an index of a non-matching selector will result in a value of undefined.

"$.baz|0" // undefined
