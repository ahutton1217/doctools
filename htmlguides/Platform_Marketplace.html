This document provides an overview of the Platform Marketplace .

Overview

The Platform Marketplace is a repository of building blocks for both Apps and APIs. Developers can discover and install components from the Marketplace, and they can also publish new components. The Platform Marketplace supports setting the access policy of a component. The following access policies are supported:

  • public: Everyone can see it (even those outside of your organization).

  • private: Only the publisher see it. This is the default policy when a component is first published.

  • org: Only certain organizations can see it. Organizations can be added or removed to/from a component via the appc org command.

  • user_or_org: Only certain organizations or users can see it. Users can be added or removed to/from a component via the appc user command, and Organizations can be added or removed to/from a component via the appc org command.

Marketplace access and publishing

You can access the Platform Marketplace in the Platform Marketplace or via the Appcelerator CLI.

To publish something to the Platform Marketplace, you must use the Appcelerator CLI.

Currently, the Platform Marketplace supports the following component types:

  • Blocks: Used for building APIs

  • Connectors: Used for building APIs

  • Models: Used for building APIs

At the time of publishing this document, we do not support mobile app modules and Allow widgets.

CLI options

The Appcelerator CLI provides a number of options for interacting with the Platform Marketplace from the command line. The table below outlines the commands that can be used:

Name

Arguments

Description

search

[search_term]

Search the Marketplace for a particular component or set of components.

install

[component_name]

Install a specific component. Execute this command from within an API project. The CLI will ensure the component is installed in the proper location (i.e., models will be installed in the models directory and connectors in the connectors directory).

publish

None

Publish a component to the Marketplace. Execute this command from the root directory of your component project.

unpublish

[component_name]

Unpublish a component from the Marketplace. The [compononent_name] is only needed when the command is executed outside the component’s project directory.

access

get or set

Get the current access policy or set it. The valid values are: public, private, org, and user_or_org. Defaults to private.

org

add, rm, or ls

Add, remove or list the current organizations tied to your component. Note: the org values are only relevant if your access policy is set to org or org_or_user.

user

add, rm, or ls

Add, remove or list the current users tied to your component. Note: the user values are only relevant if your access policy is set to user_or_org.

owner

add, rm, or ls

Add, remove or list the current owners tied to your component. Owners can set/change values for the access, org and user commands.

Install a model

To install an existing Model from the Platform Marketplace, execute the following command from within your API project: appc install model/model_name

The install command will place the model in the models directory of your project.

Install a connector

To install an existing Connector from the Platform Marketplace, execute the following command from within your API project: appc install <connector/connector_name>

The install command will place the connector in the connectors directory of your project.

Install a block

To install an existing Block from the Platform Marketplace, run the following command from within your API project: appc install <block/block_name>

The install command will place the block in the blocks directory of your project.

Searching for models, connectors, and blocks

Searching for models, connectors, and blocks can be accomplished by two different methods: using the CLI search command or searching directly in the Platform Marketplace.

If you are unsure of the exact name, you can search the Marketplace using the CLI: appc search <search_string>

CLI search suggestions

When using appc search, don't use wildcards. The search command uses a fuzzy search algorithm to return a list of components.

If you know the component name starts with a certain string or character, try the appc search sale command. This example will return the connector/appc.salesforce connector.

If you want to get a list of all the connectors, try the appc search connector command. This example will return a list of all the available connectors.
