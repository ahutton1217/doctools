About This Release

This is a General Availability (GA) release of the Appcelerator Platform including Appcelerator Studio 3.2.0 and support for Titanium SDK 3.2.0 and Alloy 1.3.

These release notes only address the enterprise features associated with the Appcelerator Platform. For updates about the underlying components of the Appcelerator Platform, see the links in the Titanium Components section below.

Breaking Changes

Appcelerator Test 6619.13 Upgrade

The Appcelerator Test 6619.13 upgrade contains some breaking changes from the previous version:

  • Support for universal mobile applications
  • In the Central tab of the Test Dashboard, Device Agents was renamed to Device Clouds, though it is still reference as Device Agents in the other parts of the GUI.

Since this upgrade, Appcelerator Test supports universal mobile applications that can be played on both Android and iOS. Previously, two separate mobile applications were created in the Test dashboard.

After you upgrade the Test module, all existing projects using Appcelerator Test will have this service disabled and you will need to migrate to the universal application.

In Appcelerator Studio, re-enable Appcelerator Service for your application, which creates the universal mobile application for you.

If you have existing test clips, you need to update your Targets (device/test clip pair) in the Test dashboard to reference the new universal mobile application.

For instructions, see the "Test Module Build 6619.13 Upgrade" section of the Test page.

Titanium Components

Appcelerator Studio requires Titanium components to develop mobile applications. By default, Appcelerator Studio automatically checks and prompts you to update the components that are GA. To manually check for updates, from the menu bar, select Help > Check for Appcelerator Updates.

This version of Studio is tested against and supports the following Titanium components:
